# Moving Development to a New Computer

*Checklist for migrating the Altianly dev environment. Written 2026-07-04.*

Everything committed to git is safe on GitHub (`github.com/vxc0812/altianly`) — the app itself needs no backup. This checklist covers what git does **not** carry.

---

## Before leaving the old computer

- [ ] **Push any pending work**: in the project folder run `git status` (should be clean) and `git log origin/master..master` (should print nothing). If anything shows up: `git add -A && git commit && git push`.
- [ ] **Copy the gitignored project files** (they are intentionally kept out of the public repo, so they exist only on this machine). From the project root, copy these into a folder on a USB drive / cloud drive:
  - `CLAUDE.md`, `AGENTS.md`, `PROJECT_DOCS.md` — Claude Code's project instructions (without these, Claude starts new sessions blind about codebase conventions)
  - `Altianly/` — Obsidian vault (plans and notes)
  - `Apple Integration/` — health-integration research docs
  - `POCKETPAL_INTEGRATION.md`, `POCKETPAL_QUICKSTART.md`, `POCKETPAL_SUMMARY.md`
  - `brand-spec.md`, `workout_liability_warning.html`
- [ ] **Copy Claude Code's home folder**: `C:\Users\<you>\.claude\` (in your user profile, NOT in the project). This carries Claude's persistent project memory (launch-plan state, working preferences, debugging notes) and your Claude Code settings.
- [ ] **Do NOT copy**: `node_modules/`, `dist/`, `.expo/`, `android/` — all regenerated by tooling. Copying them wastes hours and can break on a different machine.

## Nothing secret lives on this machine

API keys (USDA, Resend) are stored server-side as Cloudflare Worker secrets. Cloudflare/GitHub/Claude logins are per-machine OAuth sessions that don't transfer — you just log in again (next section).

---

## On the new computer

Prerequisites: install **Git**, **Node.js (LTS)**, and **Claude Code**.

```bash
# 1. Get the code
git clone https://github.com/vxc0812/altianly.git
cd altianly

# 2. Restore the carried-over files
#    - copy the gitignored files/folders (list above) into this project folder
#    - copy the saved .claude folder to C:\Users\<you>\.claude

# 3. Install dependencies
npm install

# 4. Re-authenticate (each prompts a browser login)
npx wrangler login        # Cloudflare — needed for worker deploys/secrets
git push --dry-run        # triggers GitHub sign-in via credential manager
# Claude Code: run `claude` in the project folder, then /login if prompted

# 5. Smoke test
npm run web               # app at http://localhost:8081
npm run typecheck && npm run lint
```

## Verify everything works

- [ ] `npm run web` → app loads in the browser, tabs + dashboard render
- [ ] `npx wrangler whoami` → shows your Cloudflare account
- [ ] Make a trivial change, `git push` → succeeds, and Cloudflare Pages auto-deploys it (~3 min, check altianly.com)
- [ ] Start Claude Code → it should recall project context (memory folder worked)

## Reference — what lives where

| Thing | Where it lives | Action on migration |
|---|---|---|
| App code, docs, worker | GitHub repo | `git clone` |
| Live website | Cloudflare Pages (auto-deploys from GitHub) | Nothing |
| Server + secrets (USDA, Resend keys) | Cloudflare Worker `altianly-ai` | Nothing |
| Accounts/data of app users | Cloudflare KV | Nothing |
| Domain | Registered at DreamHost, DNS on Cloudflare | Nothing |
| Claude's project instructions | Gitignored files in project root | Manual copy |
| Claude's memory + settings | `~/.claude` | Manual copy |
| Tool logins (Cloudflare, GitHub, Claude) | Per-machine OAuth | Re-login |

## Deploy commands (unchanged, for reference)

```bash
# Web: just push — Pages auto-builds from GitHub
git push

# Worker (only when workers/ai-proxy/index.js changes):
npx wrangler deploy --config workers/ai-proxy/wrangler.toml
# ⚠ always pass --config; wrangler otherwise picks up the ROOT wrangler.jsonc
```
